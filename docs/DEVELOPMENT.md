# Руководство разработчика (Developer Guide)

Добро пожаловать в команду Vibe! Этот документ поможет вам быстро настроить окружение, понять стандарты кода и правила разработки.

---

## 1. Подготовка окружения

### Требования
- Python 3.9+
- Node.js 16+ и npm

### Инициализация (Backend)
Рекомендуется использовать виртуальное окружение:
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Для Linux/Mac
venv\Scripts\activate     # Для Windows

pip install -r requirements.txt
```

### Инициализация (Frontend)
```bash
cd frontend
npm install
```

---

## 2. Стандарты написания кода

### Backend (Python)
- Используйте строгую аннотацию типов (Type Hints) во всех функциях роутов.
- Бизнес-логику старайтесь выносить из роутов в сервисы (если проект будет расти) или используйте CRUD-операции.
- Схемы (Pydantic) держите в папке `schemas/`, разделяя файлы по сущностям. Всегда создавайте отдельные модели для входных данных (`EventCreate`) и выходных (`EventResponse`).
- Документируйте новые эндпоинты, добавляя параметры `summary` и `description` в декоратор `@router.get(...)`.

### Frontend (React)
- Используйте функциональные компоненты и хуки (`useState`, `useEffect`, `useContext`).
- Файлы стилей `*.css` должны лежать рядом с компонентом, к которому они относятся. 
- Избегайте inline-стилей, используйте классы. Придерживайтесь глобальных CSS-переменных (настроены в `index.css`), таких как `var(--vibe-primary)` и `var(--glass-bg)`, для поддержания единого стиля.
- Все сетевые запросы инкапсулированы в `src/services/api.js`. Не вызывайте `axios` напрямую из UI-компонентов.

---

## 3. Работа с миграциями БД
*На данный момент в проекте не используется Alembic (создание таблиц происходит через `Base.metadata.create_all`).*
Если вы добавляете новую колонку в модель БД в `backend/app/database/`, вам придётся пересоздать базу `database_vibe.db` (или добавить миграции с помощью Alembic в будущем).

---

## 4. Написание тестов

Тестирование бэкенда выполняется с помощью `pytest`.
В качестве клиента используется `TestClient` из пакета FastAPI.

**Правила написания тестов:**
1. Тесты лежат в папке `backend/tests/`.
2. Используйте фикстуры из `conftest.py` (например, `client`, `db`, `user_token_headers`), чтобы не дублировать код авторизации и создания моковых данных.
3. База данных для тестов создаётся в памяти или во временном файле, она изолирована от реальной БД приложения!

**Запуск тестов:**
```bash
python -m pytest backend/tests/ -v
```
