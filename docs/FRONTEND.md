# Архитектура Frontend (React)

Фронтенд проекта Vibe представляет собой Single Page Application (SPA), написанное на React.js без использования громоздких фреймворков вроде Next.js, чтобы обеспечить максимальную скорость и простоту.

---

## 1. Структура проекта

Код фронтенда находится в папке `frontend/src/` и разделён на следующие модули:

- `components/` — переиспользуемые UI-компоненты (например, `Header`, `AuthModal`).
- `pages/` — компоненты-страницы, привязанные к определённым маршрутам (URL).
- `services/` — логика взаимодействия с бэкендом (API-клиент на базе Axios).
- `context/` — глобальное состояние приложения (Context API).
- `App.js` — корневой компонент, в котором настроен роутинг.
- `index.css` — глобальные стили и CSS-переменные дизайн-системы.

---

## 2. Маршрутизация (Routing)

Для навигации без перезагрузки страницы используется `react-router-dom`. Маршруты настроены в `App.js`:

| Маршрут | Компонент | Описание |
|---------|-----------|----------|
| `/` | `Home` | Главная страница (Лендинг). |
| `/events` | `EventsPage` | Список всех доступных событий с поиском и фильтрацией. |
| `/events/:id` | `EventDetailPage` | Карточка конкретного события, кнопки участия/удаления. |
| `/events/:id/manage` | `ManageParticipantsPage` | Управление заявками (только для организатора). |
| `/create` | `CreateEventPage` | Форма создания нового события. |
| `/profile` | `ProfilePage` | Профиль текущего авторизованного пользователя. |
| `/users/:id` | `ProfilePage` | Профиль любого другого пользователя (по ID). |

---

## 3. Управление состоянием (State Management)

Вместо Redux используется встроенный **React Context API**.

Главный контекст — **`AuthContext`** (`src/context/AuthContext.js`). Он оборачивает всё приложение и предоставляет:
1. `user` — данные текущего авторизованного пользователя.
2. `login(token)`, `logout()` — функции для управления JWT-токеном (сохраняется в `localStorage`).
3. `isAuthModalOpen`, `openAuthModal()`, `closeAuthModal()` — управление глобальным всплывающим окном авторизации.

Благодаря этому окну (`AuthModal`), мы смогли избавиться от отдельных страниц `/login` и `/register` — теперь пользователь может авторизоваться, не покидая текущую страницу.

---

## 4. Взаимодействие с API

Все запросы к бэкенду вынесены в отдельный сервис: `src/services/api.js`.
Мы используем библиотеку **Axios**.

**Особенности `api.js`:**
- Создан инстанс `apiClient` с базовым URL `http://localhost:8000`.
- Настроен **Interceptors** (перехватчик) запросов: он автоматически достаёт токен из `localStorage` и подставляет его в заголовок `Authorization: Bearer <token>` при каждом запросе.
- Все эндпоинты (получение профиля, создание события, удаление) обернуты в удобные асинхронные функции (например, `createEvent(eventData)`, `deleteUser(password)`).

---

## 5. Дизайн-система и Стилизация

В проекте используется подход **Vanilla CSS** (чистый CSS без Tailwind или Bootstrap). Это даёт полный контроль над стилями и позволяет реализовать эффекты **Glassmorphism** (эффект матового стекла).

- В `index.css` объявлены глобальные CSS-переменные (цвета, шрифты, градиенты, тени).
- Локальные стили для каждой страницы/компонента лежат рядом с ними (например, `ProfilePage.css`).
- Основной акцентный цвет: фиолетовый (`--vibe-primary: #a855f7`).
- Шрифты: современные системные шрифты (Inter, Roboto).

### Эффект матового стекла (Glass)
Во всём приложении активно используется класс или миксин с эффектом:
```css
background: rgba(255, 255, 255, 0.05);
backdrop-filter: blur(10px);
border: 1px solid rgba(255, 255, 255, 0.1);
```
Это придаёт интерфейсу премиальный, современный и динамичный вид.
